<div class="prose" dir="ltr"><p>Socket’s Threat Research Team uncovered a malicious Go module, <a class="chakra-link css-1dkuy37" href="https://socket.dev/go/package/github.com/xinfeisoft/crypto"><code>github[.]com/xinfeisoft/crypto</code></a>, that imitates the legitimate <a class="chakra-link css-1dkuy37" href="https://socket.dev/go/package/golang.org/x/crypto"><code>golang.org/x/crypto</code></a> codebase but inserts a backdoor in <a class="chakra-link css-1dkuy37" href="https://socket.dev/go/package/github.com/xinfeisoft/crypto?section=files&amp;version=v0.15.0&amp;path=ssh%2Fterminal%2Fterminal.go"><code>ssh/terminal/terminal.go</code></a>. That choice was strategic: <code>golang.org/x/crypto</code> is one of the Go ecosystem’s foundational cryptography codebases, maintained by the Go project and widely relied on for primitives and packages such as <code>bcrypt</code>, <code>argon2</code>, <code>chacha20</code>, and <code>ssh</code>, which makes it a high-trust impersonation target in dependency graphs.</p><p>When a victim application prompts for a password via <a class="chakra-link css-1dkuy37" href="https://socket.dev/go/package/github.com/xinfeisoft/crypto?section=files&amp;version=v0.15.0&amp;path=ssh%2Fterminal%2Fterminal.go#L52"><code>ReadPassword</code></a>, the modified function captures the secret, writes it locally, then reaches out to threat actor-controlled infrastructure for follow-on instructions. It fetches a GitHub hosted “<a class="chakra-link css-1dkuy37" href="https://socket.dev/go/package/github.com/xinfeisoft/crypto?section=files&amp;version=v0.15.0&amp;path=ssh%2Fterminal%2Fterminal.go#L55">update</a>” resource, posts the harvested password to a threat actor-supplied endpoint, retrieves a shell script from that endpoint, and executes it to run arbitrary commands on the host.</p><p>The downloaded script acts as a Linux stager. It appends a threat actor SSH key to <code>/home/ubuntu/.ssh/authorized_keys</code>, sets <code>iptables</code> default policies to <code>ACCEPT</code>, and downloads additional payloads from <code>img[.]spoolsv[.]cc</code> while disguising them with the <code>.mp5</code> extension, a media-like label that can help binaries blend in during quick review. The staged payloads include <code>sss.mp5</code> and <code>555.mp5</code>, which we analyzed and confirmed as a Rekoobe Linux backdoor.</p><p>This activity fits namespace confusion and impersonation of the legitimate <code>golang.org/x/crypto</code> subrepository (and its GitHub mirror <code>github.com/golang/crypto</code>). The legitimate project identifies <code>go.googlesource.com/crypto</code> as canonical and treats GitHub as a mirror, a distinction the threat actor abuses to make <code>github.com/xinfeisoft/crypto</code> look routine in dependency graphs.</p><p>As of this writing, the module remains listed on <code>pkg.go.dev</code>, which currently shows <code>github[.]com/xinfeisoft/crypto</code> at <code>v0.15.0</code> with a February 20, 2025 publication date. Socket was still able to fetch the malicious module from the public Go module mirror as of December 16, 2025. After we reported the package, the Go security team confirmed that the public Go module proxy now blocks it as malicious and returns a <code>403 SECURITY ERROR</code> response instead of serving it. That mitigation reduces exposure through Go’s default module resolution path, but it does not lessen the severity of a package that impersonated a foundational Go dependency, harvested passwords, and deployed a Linux backdoor chain. We appreciate the Go security team’s prompt response in this case and in prior cases where we reported malicious modules, and we are grateful for their continued work to keep the Go ecosystem safe. We also filed an abuse report requesting action on the publisher’s GitHub account, which remains live as of this writing.</p><figure class="css-1efvhw7"><img alt=" " class="css-4p5h3n" loading="lazy" The <code>crypto</code> repository publishes the malicious Go module (<code>github[.]com/xinfeisoft/crypto</code>) that backdoors <code>ssh/terminal.ReadPassword()</code> to capture interactive secrets, POST them to a threat actor-supplied endpoint, and execute server-provided shell content. The separate <code>vue-element-admin</code> repository serves as staging infrastructure. It hosts <code>public/update.html</code> on GitHub Raw, which exposes the raw contents of a repository file through a direct URL. The backdoor fetches that file at runtime to obtain the next hop (<code>img[.]spoolsv[.]cc/seed.php</code>) and bootstrap the <code>curl | sh</code> stager.
